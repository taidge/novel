# 文件嵌入

Novel 的一个关键特性是可以将外部源代码文件直接嵌入到文档中。

## 基本用法

在代码围栏上使用 `file` 属性来嵌入文件:

````markdown
```rust file="./path/to/file.rs"
```
````

代码块的主体应当留空 —— 内容会从指定的文件加载。

## 路径解析

### 相对路径

以 `./` 或 `../` 开头的路径相对于当前 Markdown 文件解析:

````markdown
```rust file="./examples/hello.rs"
```
````

### 根相对路径

使用 `<root>/` 前缀来引用项目根目录下的文件:

````markdown
```rust file="<root>/src/main.rs"
```
````

## 行号范围

你可以只嵌入文件中的特定行:

````markdown
```rust file="./examples/hello.rs#L3-L8"
```
````

支持的格式:
- `#L5` —— 单行
- `#L5-L10` —— 行范围(闭区间)
- `#5-10` —— 不带 `L` 前缀的行范围

## 在线示例

下面嵌入的是本文档站点的 Novel 配置文件:

```toml file="<root>/novel.toml"
```

## 使用场景

::: tip 何时使用文件嵌入
- **有测试的代码**: 嵌入实际被编译和测试的代码,确保文档与代码同步
- **配置示例**: 展示项目中真实的配置文件
- **API 示例**: 引入与源码一同维护的示例代码
:::

::: warning
请确保被嵌入的文件在构建时存在。缺失的文件会产生警告并在代码块中显示错误信息。
:::
