# 搜索

Novel 内置了客户端搜索,无需任何外部依赖或服务。

## 工作原理

在 `novel build` 期间,Novel 会从所有页面生成一个搜索索引(`search-index.json`)。搜索完全在客户端进行 —— 不需要服务器或第三方服务。

搜索索引包含:
- 页面标题
- 章节标题
- 页面内容(已去除 HTML)
- 页面描述

## 使用搜索

搜索框显示在每个文档页面的侧边栏中。你也可以在页面上任意位置按 `/` 键将光标聚焦到搜索输入框。

### 快捷键

| 键 | 作用 |
|-----|------|
| `/` | 聚焦到搜索输入框 |
| `Escape` | 关闭搜索结果 |
| `上下方向键` | 在结果间导航 |
| `Enter` | 跳转到选中的结果 |

## 搜索排序

结果按相关性排序:

1. **标题匹配**得分最高(10 分)
2. **章节标题匹配**得分中等(5 分)
3. **正文匹配**得分最低(1 分)

结果按得分排序,最多显示 8 条。

## 搜索预览

每条搜索结果会显示:
- 页面标题(匹配的关键词会被高亮)
- 内容预览,显示匹配项所在的上下文

## 性能

搜索索引是**懒加载**的 —— 只有当搜索输入框首次获得焦点时才会获取。这意味着搜索索引不会影响页面的初次加载性能。
